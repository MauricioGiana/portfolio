Automated Customer Onboarding & Secure AI Conversational Engine

Node.js · Python · AWS Lambda · RabbitMQ · Meta Embedded Signup · AI Agents

Challenge & Context

  • Designed an automated system enabling multi-tenant business integration as an official Meta Tech Provider, allowing clients to register their WhatsApp Business numbers autonomously via Embedded Signup.
  • Deployed 24/7 AI-driven conversational assistants capable of executing operations like automated appointment scheduling and inquiries, while addressing the critical challenge of exposing public LLMs to prompt injection, cost abuse, and malicious resource exploitation.

Architecture & Implementation

  • Serverless Webhook Gateway: Implemented AWS Lambda to ingest high-frequency webhooks and immediate messaging payloads from the Meta API with minimal operational cost and near-instant scaling.
  • Asynchronous Message Buffering: Integrated a RabbitMQ ingestion layer to decouple high-volume messaging events from core LLM processing pipelines, preventing message drops and managing downstream LLM rate limits smoothly.
  • Defensive Guardrail Architecture: Implemented dedicated guardrail layers designed to capture prompt injections, jailbreaks, and off-topic interactions before they reach the orchestration layer, minimizing exposure risks.
  • Strict Tool & Operation Scoping: Configured strict function-calling boundaries and tool limitations per session. Designed custom backend rate limiting by phone number and IP, coupled with developing operational heuristics to throttle suspicious request flows and secure sensitive downstream actions.

Impact & Results

  • Enabled friction-free, autonomous B2B user onboarding as a Tech Provider, eliminating manual setups for client business messaging lines.
  • Successfully automated 24/7 core platform interactions (appointments/FAQ) while preserving complete guardrail-backed defense over token costs, API throttling, and security boundaries.

Distributed Observability & Context Propagation

Go · TypeScript · OpenTelemetry · Jaeger · Prometheus · Elastic Cloud · Grafana

Challenge & Context

  • Legacy logging structures lacked correlation, making distributed debugging across asynchronous microservices a highly time-consuming task where engineering teams acted reactively to system failures.
  • Required a rapid, high-coverage observability rollout across existing backend workflows without introducing excessive engineering overhead or slowing down the product roadmap.

Architecture & Implementation

  • Hybrid Instrumentation Strategy: Leveraged OpenTelemetry's auto-instrumentation SDKs for rapid baseline visibility, while progressively embedding manual spans and metrics for granular system domain insight.
  • Distributed Tracing Pipeline: Configured OpenTelemetry SDKs to export distributed traces via low-overhead gRPC protocols directly to Jaeger, centralizing cross-service and asynchronous messaging validation.
  • Multi-Provider Telemetry Topologies: Deployed Prometheus as the core metrics provider to track system golden signals and fire alerts on critical infrastructure anomalies such as memory leaks or pod crashloops. Integrated logging pipelines by injecting OpenTelemetry Trace IDs into structured JSON logs using Winston (and Go standard structured loggers), streaming them directly to Elastic Cloud for unified lookup.
  • Unified Diagnostics Layer: Consolidated Prometheus metrics and Jaeger trace visualizers into centralized Grafana dashboards to serve as a single pane of glass for real-time diagnostics and agentic-ready automated debugging.

Impact & Results

  • Drastically slashed Mean Time to Resolution (MTTR) from hours to minutes by enabling engineers and AI automation workflows to jump seamlessly from an error log in Elastic Cloud to its exact correlated distributed trace graph.
  • Shifted backend system maintenance from reactive firefighting into a reliable, proactive culture driven by early anomaly alerts.

Event-Driven Third-Party Calendar Integration

Go · RabbitMQ · PostgreSQL · OAuth 2.0

Challenge & Context

  • Built a highly available, multi-tenant integration to synchronize platform appointment lifecycles with external user calendars in real time.
  • The system needed to process high volumes of concurrent operations per tenant without impacting the core platform's performance or introducing tight coupling with external APIs.

Architecture & Implementation

  • Distributed OAuth Setup: Implemented an isolated Integration Gateway service to manage secure token storage and authentication flows.
  • Event-Driven Orchestration: Deployed a RabbitMQ Topic Exchange topology. The core application publishes lifecycle events such as Created, Updated, and Cancelled, which are asynchronously consumed by the integration layer.
  • Smart State Caching: Local database schemas mirror external resource mappings, acting as an idempotent read-cache to reduce third-party API rate-limiting overhead and ensure transactional traceability.
  • Complex State Mutations: Handled complex lifecycle operations asynchronously, including resource reassignment flows that trigger combined deletion and atomic creation logic.

Why Go?

  • Selected Go for the consumer engine due to its exceptional concurrency efficiency with goroutines and a low memory footprint, preventing bottleneck risks during peak multi-tenant scheduling activity.
